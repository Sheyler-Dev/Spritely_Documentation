Getting started

Instalación

El script principal del plugin instala o actualiza el módulo Spritely en ReplicatedStorage y el loader Spritely_Loader en StarterPlayerScripts cuando detecta una versión distinta.

Runtime esperado

Tu juego debe tener game.ReplicatedStorage.Spritely y una carpeta game.ReplicatedStorage.SpriteFolder con las carpetas generadas por el editor.

Lua
local Spritely = require(game.ReplicatedStorage:WaitForChild("Spritely"))
local spriteFolder = game.ReplicatedStorage:WaitForChild("SpriteFolder")

Objetos compatibles

La API runtime trabaja con ImageLabel e ImageButton. El plugin puede marcar objetos con el tag SpritelyObject para raycast y sistemas de selección.

ImageLabel ImageButton CollectionService
Modelo mental

Conceptos

Spritely guarda sprites como carpetas con frames serializados en atributos. Al cargar un sprite, el módulo prepara buffers, contenidos estáticos y datos de colisión para cada frame.

SpriteFolder

Contenedor en ReplicatedStorage. Cada hijo representa un sprite importado o creado desde el editor.

Frame

Cada frame guarda atributo Image. El runtime lo convierte en EditableImage o contenido estático.

Type

Static usa contenidos creados por AssetService. Dynamic escribe buffers sobre el objeto actual.

CollisionType Uso Atributos relacionados
Box Área rectangular basada en los píxeles visibles. Relative
Radial Colisión circular alrededor del centro de masa. Centro de masa calculado
Precise Trazo más detallado para forma por píxeles. Resolution
Convex Hull convexo para colisiones más simples. Datos generados por el módulo de colisión
Custom Vértices editados manualmente desde el editor. VertexData
Studio plugin

Editor Plugin

El plugin crea una toolbar llamada Spritely y un DockWidget para administrar sprites, frames, importación por asset id, edición de pixels, selección y mapas de colisión.

Flujo de creación

  1. Crear un sprite o importar una imagen desde asset id.
  2. Definir tamaño, frames y carpeta dentro de SpriteFolder.
  3. Ajustar CollisionType, resolución y modo relativo.
  4. Usar el módulo runtime para cargar el sprite en un ImageLabel o ImageButton.

Herramientas detectadas

  • ResizeBuffer
  • Pintar
  • Select
  • Copy_Paste
  • GenerateColliderEditor
  • PathEditorModule
  • SaveSprites
  • ImageGenerator

Debug: el plugin usa workspace.Debug_Collision_Spritely. Cuando está activo, el runtime dibuja visuales de colisión sobre el objeto cargado.

Ejemplos

Estos snippets asumen que el editor ya generó una carpeta de sprite llamada HeroIdle dentro de ReplicatedStorage.SpriteFolder.

Cargar y animar

LocalScript
local Spritely = require(game.ReplicatedStorage:WaitForChild("Spritely"))

local image = script.Parent:WaitForChild("Hero")

Spritely:CreateObject(image, "HeroIdle", 1)
Spritely:PlayAnimation(image, 12, true, 1, 8)

Colisión y raycast

Lua
local touching = Spritely.IsTouching(playerSprite, coinSprite, "Detailed")

local hit = Spritely:RayCast(
  Vector2.new(100, 120),
  Vector2.new(500, 0),
  { RespectVisible = true, RayCastType = "Exclude", Instances = {} }
)

if hit then
  print(hit.Instance, hit.Position, hit.Distance)
end
ModuleScript Spritely

Referencia API

Superficie pública extraída del módulo Spritely. Los nombres se mantienen como aparecen en el código fuente para evitar sorpresas al copiar ejemplos.

Carga Método

Spritely:CreateObject(ImageObject, SpriteFolder, Frame?)

Crea el cache de un sprite, asigna el frame inicial al objeto y registra la instancia como cargada.

ParámetroTipoDescripción
ImageObjectImageLabel | ImageButtonObjeto UI que recibirá el contenido del sprite.
SpriteFolderstringNombre de la carpeta dentro de ReplicatedStorage.SpriteFolder.
Framenumber?Frame inicial. Si no se define, usa 1.

Internamente llama GenerateObject.SetPixelHash(..., "Static"), guarda datos en AnimationsFolder y dispara InstanceLoaded.

Lua
Spritely:CreateObject(myImageLabel, "Explosion", 1)
Estado Yield

Spritely:WaitForInstanceLoaded(ImageObject, TimeOut?)

Espera hasta que un objeto haya sido registrado por CreateObject o hasta que se cumpla el tiempo máximo.

RetornoDescripción
numberTiempo transcurrido desde que empezó la espera. Si ya estaba cargado, retorna 0.

El timeout por defecto es 10 segundos. El código fuente usa Spritely.IstanceLoaded, con ese nombre exacto.

Lua
local elapsed = Spritely:WaitForInstanceLoaded(heroImage, 5)
print("Hero listo en", elapsed)
Buffers EditableImage

Spritely:GetImageBuffer(SpriteFolder, Frame)

Obtiene el buffer de pixeles de un frame y el tamaño original del sprite.

MétodoRetornoUso
GetImageBufferbuffer, Vector2Lee GenerateObject.CreatedSprites[folder][frame] y GenerateObject.Sizes[folder].
SetImageBuffernilEscribe un buffer sobre ImageObject.ImageContent.Object.
Lua
local frameBuffer, size = Spritely:GetImageBuffer("HeroIdle", 3)
Spritely:SetImageBuffer(heroImage, frameBuffer)
Colisiones Fast | Detailed

Spritely.IsTouching(Image1, Image2, Type?)

Calcula intersección entre dos sprites cargados usando sus colliders del frame actual.

TypeRetornoDescripción
Fastboolean, Vector2?, number?, string?, any?Ruta más rápida con AABB rotado y SAT/contactos resumidos.
Detailedboolean, Vector2?, number?, string?, table?Devuelve datos de contacto más completos para polígonos y radiales.

Si Type no es válido, el módulo avisa con warn y retorna false.

Lua
local hit, normal, depth, kind, contacts = Spritely.IsTouching(a, b, "Detailed")
if hit then
  print(normal, depth, kind)
end
Raycast 2D CollectionService

Spritely:RayCast(Position, Direction, Options?)

Lanza un rayo en pantalla contra objetos tagueados como SpritelyObject.

CampoTipoDescripción
RespectVisiblebooleanSi es true, ignora objetos no visibles.
RayCastType"Exclude"Modo usado por el código actual para saltar instancias listadas.
Instances{Instance}Objetos que deben excluirse si RayCastType es Exclude.

Retorna nil o una tabla con Instance, Position, Normal y Distance.

Lua
local hit = Spritely:RayCast(origin, direction, {
  RespectVisible = true,
  RayCastType = "Exclude",
  Instances = { playerSprite }
})
Audio Sound

Spritely:AddSound(SoundId, ImageObject)

Crea un Sound asociado a un Attachment en la cámara y actualiza su posición según el centro del objeto UI.

ParámetroTipoDescripción
SoundIdnumber | stringNúmero de asset o cadena completa rbxassetid://....
ImageObjectImageLabel | ImageButtonObjeto que controla la posición normalizada del sonido.
Lua
local sound = Spritely:AddSound(1234567890, coinImage)
sound.Volume = 0.5
sound:Play()
Frames Static | Dynamic

Spritely:SetFrame(ImageObject, Frame)

Cambia el frame actual del sprite cargado y actualiza el atributo Frame.

En modo Static asigna ImageContent al contenido cacheado. En modo Dynamic escribe el buffer del frame sobre el EditableImage existente.

Lua
Spritely:SetFrame(heroImage, 4)
Animación RenderStepped

Spritely:PlayAnimation(ImageObject, FPS?, Loop, MinFrame, MaxFrame)

Reproduce frames usando RunService.RenderStepped. El FPS se limita entre 1 y 120.

MétodoDescripción
PlayAnimationInicia animación desde MinFrame hasta MaxFrame; si Loop es falso, dispara Ended.
StopAnimationDesconecta las conexiones activas del objeto.
GetAnimationEventRetorna señales Played, Ended, FrameChanged y Stopped.
Lua
local events = Spritely:GetAnimationEvent(heroImage)
events.FrameChanged:Connect(function(frame)
  print("Frame", frame)
end)

Spritely:PlayAnimation(heroImage, 16, false, 1, 12)

Deploy

Google Cloud

La página es estática: index.html más assets/spritely-mark.png. Puedes publicarla con Firebase Hosting o con un bucket de Cloud Storage configurado como sitio.

Firebase Hosting

Terminal
firebase init hosting
firebase deploy --only hosting

Cloud Storage

Terminal
gcloud storage cp --recursive ./spritely-docs gs://TU_BUCKET
gcloud storage buckets update gs://TU_BUCKET --web-main-page-suffix=index.html --web-error-page=404.html
